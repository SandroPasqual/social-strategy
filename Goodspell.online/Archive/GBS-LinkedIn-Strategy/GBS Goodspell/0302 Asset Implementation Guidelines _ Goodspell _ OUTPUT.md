# 0302 Asset Implementation Guidelines | Goodspell | OUTPUT
## 030201  PAPO
### Purpose
Explains how Goodspell shows up across everyday assets — website, documents, presentations, LinkedIn, and collateral — using the visual identity and messaging defined in Phases 01–03. Provides simple, ready-to-use rules so branded materials can be created quickly without turning every task into a new strategy exercise.
### Audience
Sandro Pasqual as primary brand executor at launch. Any designer, developer, or agency who creates or updates Goodspell materials. The Logic Manager when onboarded. External collaborators receiving a production brief.
### Process
Shared after 0301 Visual Identity System is approved and 030110 Key Applications is confirmed. Execution starts from aligned templates and standards — not from scratch. The document is organized around four areas: Implementation Logic, Core Asset Directory, Application Guidelines, and Consistency Checklist.
### Outcome
A practical working set of guidelines that shortens production time, lowers the cognitive load of everyday design decisions, and keeps execution aligned with the approved visual and strategic system. The operational bridge between strategy, identity, and real-world materials.
## 030202  Implementation Logic
### Strategic Guardrails
These are the principles that keep every Goodspell asset aligned with the real weight of the work. They are not aesthetic preferences — they are structural constraints.

### Time Protection Rules
The following rules reduce the time cost of everyday brand execution.
Never start a new asset from scratch. Start from the approved template in 030110 Key Applications. Modify content — do not modify structure.
Never choose a color manually. Use the token. If the token does not cover the use case, add a new token through the intake process in 030111-08 — do not use a free value.
Never introduce a new typeface or weight not in 030106 Typography System. If a use case is not covered, document it and update the system — do not resolve it ad hoc.
Never place a logo without checking the clear space and minimum size rules in 030107 Logo System. A logo placed incorrectly is not a minor error — it is a credibility signal.
Never approve a client-facing asset that has not passed the Consistency Checklist in 030205.

## 030206  The GBS Framework as Ongoing Reference
### Agency and Designer Briefs
When briefing an external creative contributor, share this document and 030110 Key Applications as the brief. Do not translate the brief into casual language — the precision of the document is the expectation-setter. A contributor who cannot work from this brief is not the right contributor.
### Internal Alignment
When onboarding the Logic Manager or any team member with brand-execution responsibilities, this document and the full 030 VIS package are the training materials. Not a summary. Not a slide deck. The system itself.
### Evaluating New Initiatives
Before adding a new format, channel, or campaign to the Goodspell system, apply this filter:
Does it require introducing a new color, font, or visual element outside the approved system? If yes — build the governance case first.
Does it reinforce the strategic positioning defined in 0108, or does it dilute it by reaching a different audience or communicating a different value? If it dilutes — do not proceed.
Can it be produced from existing templates and tokens? If not — build the template before producing the asset.
The GBS Framework is not a constraint on growth. It is the infrastructure that makes growth manageable without degrading the brand. A new format that fits the system costs nothing in brand equity. A new format that breaks the system costs more than the format is worth.
